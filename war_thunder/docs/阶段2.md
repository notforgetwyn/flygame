# 阶段 2：工程优化 - UI组件化与场景重构

**当前阶段目标**：
- 抽离场景公共接口和基础 UI 组件，减少场景间重复代码
- 修复按键响应问题，增加继续游戏和设置页面

**本阶段完成功能**：
- 新增 BaseScene 统一场景接口
- 新增 TextBlock 统一文本渲染组件
- 新增 MenuList 统一菜单组件
- 新增 SettingsService 设置服务（JSON持久化）
- 新增 SaveService 存档服务（自动保存/读档）
- 修复继续游戏入口（有存档/无存档处理）
- 新增设置页面（难度调整：敌机速度、生成间隔）
- 统一使用 arcade.View 架构
- 游戏自动存档（按ESC或游戏结束时）

**本阶段页面与 UI 完成情况**：

主菜单页 `MenuScene`：
- 页面标题：`战争飞机雷霆`，金色大字居中
- 使用 MenuList 组件渲染四个菜单项
- 开始游戏：清空存档进入新游戏
- 继续游戏：检查存档后进入继续或提示页
- 设置：进入设置页面
- 退出游戏：退出程序
- W/S 与方向键上下切换选项，橙色高亮选中项

设置页 `SettingsScene`：
- 页面标题：设置
- 难度设置项：敌机速度(1-10)、生成间隔(0.3-3.0秒)
- A/D 或 增减按钮调整数值
- Enter 保存设置到 config/settings.json，显示"设置已保存"
- ESC 返回主菜单
- 状态提示：保存后显示绿色提示

继续游戏页 `ContinueScene`：
- 有存档：显示"有存档"、分数、关卡，Enter继续
- 无存档：显示"暂无存档"红色提示，Enter返回
- ESC 返回主菜单

游戏页 `GameScene`：
- HUD：分数、关卡、血条(左上角)
- 玩家战机：青色三角形战斗机
- 敌机：红色圆形
- 子弹：黄色矩形
- 底部操作按键提示
- 游戏结束显示红色大字

**本阶段完成设计**：
- 新增 BaseScene：统一 handle_event()、update()、render() 接口
- 新增 TextBlock：统一文本渲染，计算居中位置，支持标题/正文/提示三种字号
- 新增 MenuList：统一菜单项渲染、高亮" > 选项 < "、上下移动、键盘处理
- MenuScene/SettingsScene/ContinueScene 复用公共组件
- SettingsService：单例模式，load/save自动创建目录和默认值
- SaveService：静态方法，save/load/clear/has_save，自动创建目录
- 输入处理统一使用 KEYDOWN 事件驱动

**本阶段数据流设计**：
- 主菜单继续游戏：MenuScene -> SaveService.has_save() -> 有存档SaveService.load()进入GameScene，无存档进入ContinueScene提示
- 设置保存：SettingsScene -> SettingsService.load() -> 修改settings -> Enter调用save()写入settings.json
- 游戏存档：GameScene.on_update() -> SaveService.save(self.save_state()) 实时保存
- 游戏读档：MenuScene继续游戏 -> ContinueScene -> GameScene.load_state(data)
- 游戏结束：SaveService.clear() 清除无效存档

**本阶段核心业务场景完成进度**：
- 主菜单业务场景：已完成（开始游戏/继续游戏/设置/退出）
- 设置业务场景：已完成（难度调整/保存/返回）
- 存档业务场景：已完成（自动存档/读档/无存档提示）
- UI 组件化业务场景：已完成（TextBlock/MenuList抽离）
