# 阶段 2：工程优化第一小步

**当前阶段目标**：
- 抽离场景公共接口和基础 UI 组件，减少场景间重复代码
- 修复按键响应问题，增加继续游戏和设置页面

**本阶段完成功能**：
- 新增 BaseScene 统一场景接口
- 新增 TextBlock 统一文本渲染
- 新增 MenuList 统一菜单组件
- 修复继续游戏入口（有存档/无存档处理）
- 新增设置页面（难度调整）
- 统一使用 KEYDOWN 处理输入

**本阶段页面与 UI 完成情况**：

主菜单页 `MenuScene`：
- 使用 MenuList 组件渲染四个菜单项
- 开始游戏：清空存档进入新游戏
- 继续游戏：检查存档后进入继续或提示页
- 设置：进入设置页面
- 退出游戏：退出程序
- W/S 与方向键切换选项

设置页 `SettingsScene`：
- 页面标题：设置
- 难度设置项：敌机速度、生成频率
- A/D 调整数值
- Enter 保存设置到 config/settings.json
- ESC 返回主菜单

继续游戏页 `ContinueScene`：
- 有存档：显示存档信息，进入游戏
- 无存档：显示提示，返回主菜单

游戏页 `GameView`：
- 保持原有功能
- 按键响应优化

**本阶段完成设计**：
- 新增 BaseScene：统一 handle_event()、update()、render() 接口
- 新增 TextBlock：统一文本渲染，计算居中位置
- 新增 MenuList：统一菜单项渲染、高亮、上下移动
- MenuScene/SettingsScene/ContinueScene 复用公共组件
- 输入处理统一使用 KEYDOWN

**本阶段数据流设计**：
- 主菜单继续游戏：MenuScene -> SaveService.load() -> 有存档进入GameView，无存档进入提示页
- 设置保存：SettingsScene -> SettingsService.save() -> 写入settings.json

**本阶段核心业务场景完成进度**：
- 主菜单业务场景：已完成
- 设置业务场景：已完成
- 存档业务场景：已完成
- UI 组件化业务场景：已完成文本和菜单组件
